Document Objective

This document explains the main architectural principles that will guide the implementation of the MyTestCaseProject software. It’s an extraction of the Software Architecture and Design Document with a focus on the development choices that every developpers need to understand.

System physical architecture overview

The MyTestCaseProject system is basically made of:

  • An instrument (hardware and firmware parts).

  • A computer (PC under Windows 7) running the MyTestCaseProject software. Basically, the main role of this software is to provide a user interface for the end user, allowing the user to use the system. The instrument and the <XXX> software are connected through a TCP/IP interface.

  • Peripherals (Hand scanner for barcode inputs, printer).
